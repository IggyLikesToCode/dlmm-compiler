Fix NNLS Strategy Truncation Algorithm
Problem
In src/python/templates.py lines 69-78, when max_strategies is set:

if max_strategies is not None and len(nonzero) > max_strategies:
    top_k_in_nonzero = np.argsort(weights[nonzero])[::-1][:max_strategies]

This takes the top-k strategies by weight and keeps their original weights. But the optimal k-strategy solution may use entirely different templates or different weight ratios than the top-k from an N-strategy solution.

Solution: Re-solve NNLS with Reduced Template Set
After identifying candidate strategies, re-run NNLS with only those templates to find the truly optimal weights for that subset.

Implementation Steps
Step 1: Refactor approximate_nnls() to re-optimize

Replace the current truncation logic with:

Identify the top-k templates by weight (as now)
Create a reduced template matrix with only those k templates
Re-run nnls() on the reduced matrix to get optimal weights
Return the re-optimized result
Step 2: Add optional iterative refinement

After re-solving, check if any zero-weight strategies should be swapped out:

If a strategy got weight ~0 after re-optimization, it wasn't a good pick
Could try replacing it with the next-best candidate from original solution
Step 3: Add quality metrics comparison

Log/return both:

R-squared of full solution (before truncation)
R-squared of truncated solution (after re-optimization)
This helps users understand the cost of limiting strategies.
